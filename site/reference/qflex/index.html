    ./qflex

    A CLI tool to manage QFlex operations. (suggest running ./qflex --show-completion >> ~/.bashrc && source ~/.bashrc to enable autocompletions)

    Usage:

    ./qflex [OPTIONS] COMMAND [ARGS]...

    Options:

      --install-completion  Install completion for the current shell.
  --show-completion     Show completion for the current shell, to copy it or
                        customize the installation.
  --help                Show this message and exit.

    boot

    Boot a QEMU virtual machine with specified parameters. Once booted, you can log in with username 'root' and no password. After installing your desired packages and making changes, you can save the state with Ctrl+A, then C to create a snapshot named 'base' by savevm base, then exit with Ctrl+C, then quit. This is the second step to create the images for an experiment, with all the necessary packages installed.

    Usage:

    ./qflex boot [OPTIONS]

    Options:

      --core-count INTEGER            Number of CPU cores for the VM.  \[required]
  --double-cores / --no-double-cores
                                  Double the number of CPU cores for the
                                  client.  \[required]
  --quantum-size-ns INTEGER       Quantum size for the simulator in
                                  nanoseconds.  \[required]
  --llc-size-per-tile-mb INTEGER  LLC size per tile in MB.  \[required]
  --parallel / --no-parallel      Whether the simulation is parallel or not.
                                  \[required]
  --network TEXT                  Network mode, either user or none
                                  \[required]
  --memory-gb INTEGER             Memory size for the VM in GB.  \[required]
  --host-name TEXT                Host name, used to create initial ipns file
                                  \[required]
  --workload-name TEXT            Workload name  \[required]
  --population-seconds FLOAT      Population size for the workload in seconds.
                                  \[required]
  --sample-size INTEGER           Sample size for the workload.  \[required]
  --consolidated / --no-consolidated
                                  Whether the workload is consolidated or not.
                                  \[required]
  --primary-ipc FLOAT             Target IPC for primary workload.
                                  \[required]
  --primary-core-start INTEGER    Starting core for primary workload.
                                  \[required]
  --secondary-core-start INTEGER  Starting core for secondary workload. Only
                                  used if consolidated is True.  \[default:
                                  -1]
  --secondary-ipc FLOAT           Target IPC for secondary workload. Only used
                                  if consolidated is True.  \[default: 0.0]
  --phantom-cpu-ipc FLOAT         Target IPC for phantom CPU. This is used for
                                  the client in the same node. Only used in
                                  double core mode.  \[default: -1.0]
  --experiment-name TEXT          Name of the experiment. Used for organizing
                                  output files.  \[default: default-
                                  experiment]
  --image-name TEXT               Name of the image file to load.  \[default:
                                  root.qcow2]
  --image-folder TEXT             Folder where images are stored.  \[default:
                                  ./images]
  --unique / --no-unique          Whether to keep the experiment folder unique
                                  by adding a timestamp.  \[default: unique]
  --use-image-directly / --no-use-image-directly
                                  Whether to use the image directly from the
                                  image folder or copy it to the experiment
                                  folder.  \[default: no-use-image-directly]
  --loadvm-name TEXT              Name of the loadvm to use in QEMU, optional
  --working-directory TEXT        Working directory where the experiment
                                  folders will be created.  \[default: .]
  --check-period-quantum-coeff INTEGER
                                  Coefficient to determine the check period
                                  based on quantum size. The value multiplied
                                  by quantum size to get check period.
                                  \[default: 53]
  --use-cd-rom / --no-use-cd-rom  Whether to use a CD-ROM for initial setup.
                                  \[default: no-use-cd-rom]
  --help                          Show this message and exit.

    create-base-image

    creates a base image for qemu using alpine in the images folder in root, can be used in next steps

    Usage:

    ./qflex create-base-image [OPTIONS]

    Options:

      --image-folder TEXT      Where to create the image  \[default: .]
  --image-name TEXT        Name of the base image file to create.  \[default:
                           root.qcow2]
  --image-size-gb INTEGER  Size of the base image in GB.  \[default: 8]
  --help                   Show this message and exit.

    partition

    This command partitions the functional warmup checkpoints into smaller segments for detailed (timing) simulation. It also generates the necessary scripts to run the detailed simulations.

    Usage:

    ./qflex partition [OPTIONS]

    Options:

      --experiment-name TEXT       Name of the experiment to partition the
                               checkpoints for.  \[default:
                               default_experiment]
  --working-directory TEXT     Working directory where the experiment folders
                               are located.  \[default: .]
  --partition-count INTEGER    Number of partitions to create.  \[default: 16]
  --warming-ratio INTEGER      Ratio of warming instructions before
                               measurement.  \[default: 2]
  --measurement-ratio INTEGER  Ratio of timing measurement instructions.
                               \[default: 1]
  --help                       Show this message and exit.
